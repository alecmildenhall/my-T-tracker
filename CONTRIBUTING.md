# Contributing to the HRT Shot Tracker

Thank you for your interest in contributing!  
Because this project relates to transgender health data and user safety, special
care is taken to protect this codebase from misuse.

## Who may contribute
Only **approved individual contributors** may submit pull requests.

If you are not yet approved, please open an Issue introducing yourself, what you
would like to contribute, and why. If approved, you will be invited as a
collaborator.

## Requirements for contributing
All contributors must:

1. Read and agree to the `LICENSE` file.
2. Read and agree to the `CLA.md` (Contributor License Agreement).
3. Follow the `CODE_OF_CONDUCT.md`.

Once approved, you will be added with appropriate GitHub permissions.

## Pull Request Workflow
- Forking the repo is allowed only for the purpose of submitting a PR.
- Your fork must remain **private** unless otherwise approved.
- Submit PRs against the `main` branch.
- All PRs require review and approval before merging.
- PRs must not introduce any collection of personally identifying information
  (PII) or weaken user privacy.

## Data Safety Rules (Important)
- Do not add analytics, telemetry, or logging without explicit discussion.
- Do not collect usernames, legal names, emails, or geolocation.
- Do not enable third-party SDKs or external data pipes without approval.

Our goal is to maintain a privacy-first, trans-safe app ecosystem.

Thank you for helping make this project safe and accessible!
